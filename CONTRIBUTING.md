# Contributing to AI Skills Playbook

Thank you for your interest in contributing to this book! 

## How to Contribute

### Reporting Issues

- **Typos/Errors**: Open an issue with the chapter name and description
- **Broken Links**: Reference the specific link and suggested fix
- **Outdated Content**: Describe what's outdated and current information

### Suggesting Content

- **New Skills**: Suggest skills to cover via discussions
- **New Chapters**: Propose new chapter topics
- **Examples**: Share real-world usage examples

### Pull Requests

1. Fork the repository
2. Create a branch (`git checkout -b improve-chapter-15`)
3. Make changes
4. Commit (`git commit -m 'Improve DevOps chapter'`)
5. Push (`git push origin improve-chapter-15`)
6. Open a Pull Request

## Style Guide

### Chapter Format

Each chapter should follow this structure:

```markdown
# Chapter N: Title

**Last Updated:** [Date]

---

## Overview
[Introduction to the topic]

### Skills Covered in This Chapter
[Table of skills]

---

## N.1 Skill Name with @skill-name

### Skill Introduction
[Purpose, when to use, key strengths]

---

### Skill Prompts

#### Prompt Title
**Context:** [Description]

```text
@skill-name Prompt content...
```

**Expected Output:** [Description]

---

## Best Practices Summary
[Categorized best practices]

---

## Reflection Points
[Questions for the reader]

---

**Next Chapter:** [Link]
```

### Writing Style

- Use clear, concise language
- Include practical examples
- Provide context for prompts
- Add best practices and reflection points

## Code of Conduct

Be respectful, constructive, and inclusive.

## Questions?

Open a discussion or issue. We're happy to help!
