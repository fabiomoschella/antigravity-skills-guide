# Chapter 49: Project Management

**Last Updated:** February 5, 2026

---

## Overview

Effective project management ensures software projects deliver value on time and within budget. Whether using Agile, Scrum, or other methodologies, strong project management skills help teams stay aligned, overcome obstacles, and achieve their goals.

This chapter covers essential skills for planning, tracking, and leading technical projects.

## 49.1 Agile Practices with @agile-coach

### Skill Introduction

The `@agile-coach` skill provides expertise in Agile methodologies and practices. It helps you implement and improve Scrum, Kanban, and other Agile frameworks for your team.

**When to use this skill:**
- Adopting Agile practices
- Improving sprint effectiveness
- Retrospective facilitation
- Agile transformation

**Key strengths:**
- Deep Agile knowledge
- Ceremony facilitation
- Continuous improvement

---

### Agile Coaching Prompts

#### Improving Sprint Effectiveness

**Context:** Your team's sprints consistently fail to meet commitments.

```text
@agile-coach Help improve our sprint effectiveness:

Current problems:
- Only 40-60% of committed work completed
- Scope creep mid-sprint
- Stories frequently spill over
- Team morale declining
- Stakeholders losing trust

Team details:
- 6 developers, 2 QA, 1 PM
- 2-week sprints
- Mix of feature work and support
- Using Jira for tracking

Sprint patterns:
- Overcommitment at planning
- Underestimated complexity
- Interruptions from support
- Last-minute requirements changes

Please provide:
1. Sprint health assessment
2. Estimation improvement techniques
3. Scope protection strategies
4. Handling support interruptions
5. Story refinement process
6. Definition of Ready/Done
7. Velocity stabilization
8. Stakeholder communication
```

**Expected Output:** Sprint improvement plan.

---

#### Running Better Retrospectives

```text
Please provide:
1. Retrospective format variety
2. Engagement techniques
3. Action item follow-through
4. Remote retrospective tips
5. Deeper issue exploration
6. Celebration incorporation
7. Frequency considerations
8. Anti-pattern avoidance
```

**Expected Output:** Retrospective improvement guide.

---

### Project Planning Prompts

#### Creating Technical Roadmap

**Context:** You need to plan a major platform modernization project.

```text
@project-planner Create a project roadmap for platform modernization:

Project scope:
- Migrate monolith to microservices
- Update tech stack (Node.js 14 to 20, React 16 to 18)
- Move from datacenter to cloud (AWS)
- Improve CI/CD pipeline
- Add observability

Constraints:
- 12-month timeline
- Team of 8 engineers
- Must maintain production stability
- Some features still being developed
- Budget: $800K

Dependencies:
- Security review for cloud
- Database team availability
- Training for new technologies

Please provide:
1. Project phase breakdown
2. Milestone definition
3. Work stream organization
4. Dependency mapping
5. Resource allocation
6. Risk-adjusted timeline
7. Checkpoint and go/no-go criteria
8. Communication plan
```

**Expected Output:** Detailed project roadmap.

### Risk Management Prompts

#### Project Risk Assessment

**Context:** You need to assess and plan for risks in a major project.

```text
@risk-manager Conduct risk assessment for our project:

Project overview:
- Cloud migration of core platform
- 9-month timeline
- 10-person team
- First cloud project for team

Known concerns:
- Team lacks cloud experience
- Third-party vendor dependency
- Data migration complexity
- Production cutover risk
- Budget constraints

Business context:
- Critical system (revenue generating)
- No tolerance for extended downtime
- Regulatory requirements
- Customer commitments

Please provide:
1. Risk identification framework
2. Risk categorization
3. Probability and impact assessment
4. Risk prioritization matrix
5. Mitigation strategies per risk
6. Contingency plans
7. Risk monitoring approach
8. Escalation criteria
```

**Expected Output:** Comprehensive risk management plan.

## Best Practices Summary

### Agile
1. **Sustainable Pace:** Don't overcommit
2. **Transparency:** Visible progress
3. **Adaptation:** Respond to change
4. **Retrospection:** Continuous improvement
5. **Collaboration:** Team over individuals

### Planning
1. **Iterate:** Plan progressively
2. **Buffer:** Include contingency
3. **Dependencies:** Map and manage
4. **Checkpoints:** Regular assessment
5. **Flexibility:** Adapt as needed

### Stakeholders
1. **Early Engagement:** Involve early
2. **Regular Updates:** Consistent communication
3. **No Surprises:** Flag issues early
4. **Manage Up:** Educate on realities
5. **Build Trust:** Deliver on commitments
